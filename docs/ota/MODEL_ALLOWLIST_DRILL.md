# Signed model allow-list drill (EP-06 / M1)

The node step that makes the parko `KIRRA_MODEL_ALLOWLIST` a **signed fact**
end to end: a repository publishes Uptane metadata whose `targets` entries ARE
the authorized ML model artifacts; the node verifies it through the full
fail-closed chain (role separation, freshness, rollback floor, no
mix-and-match — `kirra_release_token::uptane::verify_update` over the durable
WP-14 `UptaneTrustStore`) and emits the env file its co-located parko process
launches under. A refused manifest emits **nothing** — the previous env file
survives untouched and the CLI exits non-zero.

The mechanism (`kirra_ota_installer::model_allowlist` + the two
`kirra-ota-ctl` subcommands) is unit-tested end to end; this drill exercises
the operator-visible flow on a node.

Two-process wiring:

```text
  VERIFIER / RELEASE SIDE                      NODE SIDE
  ───────────────────────                      ─────────
  author root + role keys                      trust-provision (once)
  sign targets/snapshot/timestamp   ──file──►  model-allowlist
  (the targets ARE the model digests)            │ verify_update (fail-closed)
                                                 │ advance rollback floor (durable)
                                                 ▼
                                               /etc/kirra/model-allowlist.env
                                                 │ systemd EnvironmentFile=
                                                 ▼
                                               parko ModelAllowList (STRICT)
```

> How the metadata bundle lands on the node (file drop, `curl`, a config
> channel) is the transport's concern — the node verifies the bytes it is
> handed and never trusts the carrier. The full repository client (poll +
> fetch) is the recorded EP-13 follow-up.

---

## 0. Anchor the node's trust (once)

The node is provisioned with the repository's self-signed root
(`author_initial_root` output, as JSON). This is a one-time step — a second
`trust-provision` refuses (rotations flow through the metadata channel):

```sh
sudo kirra-ota-ctl trust-provision \
  --root /path/to/signed-root.json \
  --trust-store /var/lib/kirra/uptane-trust.json
```

## 1. Derive the allow-list from a published bundle

The bundle is one JSON document carrying the three role metadatas + signatures
(`ModelMetadataBundle`): `timestamp`/`timestamp_sig_b64`, `snapshot`/
`snapshot_sig_b64`, `targets`/`targets_sig_b64`.

```sh
sudo kirra-ota-ctl model-allowlist \
  --metadata /var/lib/kirra/model-metadata.json \
  --out /etc/kirra/model-allowlist.env \
  --trust-store /var/lib/kirra/uptane-trust.json
# → model allow-list written: /etc/kirra/model-allowlist.env (2 artifact(s), targets v5, STRICT)
```

The emitted file (atomic temp-fsync-rename; never observed torn):

```text
# Generated by kirra-ota-ctl model-allowlist — DO NOT EDIT.
# Signed model allow-list derived from verified Uptane targets v5 (2 artifact(s)).
# STRICT is pinned: an empty allow-list denies every model (deny-by-default).
KIRRA_MODEL_ALLOWLIST=<digest1>,<digest2>
KIRRA_MODEL_ALLOWLIST_STRICT=1
```

`KIRRA_MODEL_ALLOWLIST_STRICT=1` is pinned in the same artifact deliberately:
without strict, an EMPTY allow-list means parko enforcement is OFF — an empty
signed manifest would silently fail open. With strict pinned, "the manifest
authorizes nothing" is deny-every-model.

## 2. Launch parko under it

In the parko unit:

```ini
[Service]
EnvironmentFile=/etc/kirra/model-allowlist.env
```

then `systemctl restart` the unit. Every `load_model` now verifies the model
file's SHA-256 against the signed set (`verify_model_file`, #G16) — an
unlisted or drifted artifact is refused before a session is ever built.

## 3. Prove the fail-closed edges

- **Tamper**: flip one digest byte inside the bundle's `targets` after signing
  → `uptane verification refused: SignatureInvalid(Targets)`, exit non-zero,
  the previous env file untouched.
- **Replay / rollback**: re-present the same (or older) bundle → refused
  (`RollbackAttempt`) — the trust store's version floor advanced durably on
  first adoption. Publish a higher-version set to update again.
- **Expiry / freeze**: a bundle past any role's `expires_at_ms` → refused
  (`Expired`) — a frozen repository cannot pin the fleet to stale policy
  forever.
- **No trust state**: `model-allowlist` before `trust-provision` → refused
  (the node must not derive policy against unknown trust).
- **Empty manifest**: verifies fine and emits the strict-deny config — parko
  runs no model until a manifest authorizes one.

All five edges are pinned by tests: `model_allowlist::tests` (library) and
`model_allowlist_cli_tests` (CLI) in `crates/kirra-ota-installer`.
