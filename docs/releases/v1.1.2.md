# Kirra Runtime SDK v1.1.2

**Release Date:** 2026-05-27

## Summary

v1.1.2 delivers the Parko multi-crate behavioral safety workspace, full RSS (Responsibility-Sensitive Safety) integration per IEEE 2846-2022, a critical authority model correction for the LockedOut state, and aarch64 cross-compilation support. The release adds 340 tests (321 unit + 19 integration) across kirra-runtime-sdk, and 61 tests (54 unit + 7 proptests / 70,000 total cases) across the Parko workspace.

---

## Critical Fix: LockedOut Authority Model (PARK-003)

**LockedOut is a hard stop (0.0 m/s), not an MRC cap.** This corrects a semantic error that existed in the governor implementation, property tests, decision log, and backlog prompts.

- `KirraGovernor::evaluate()` on LockedOut now returns `EnforcementAction::Deny` with 0.0
- Proptest block 3 asserts `== 0.0`, not `<= 5.0`
- ADL-001 and ADL-005 updated to reflect the correct three-tier authority model:
  - **LockedOut** → 0.0 hard stop (human reset required)
  - **Degraded** → MRC cap at `MRC_VELOCITY_CEILING_MPS` (5.0 m/s)
  - **Nominal** → full kinematic envelope (35.0 m/s ceiling)

## Parko Workspace — Behavioral Safety Governor

### RSS Integration (IEEE 2846-2022)

**PARK-013 — Longitudinal RSS safe-distance**
- `longitudinal_safe_distance()` in `parko-core/src/rss.rs`
- IEEE 2846-2022 §5.1 formula; expected values computed as exact rational fractions (487/48, 142/3)

**PARK-014 — Lateral RSS safe-distance**
- `lateral_stop_distance()` closure; negating both velocities produces identical margin (`.abs()` contract verified)

**PARK-015 — RssState wired into posture engine**
- `AppState`: `rss_active_violation: Arc<AtomicBool>` + `rss_recovery_streak`
- `PostureRecalcTrigger::RssViolation(RssState)` added
- Recovery hysteresis: 5-streak within 10 s window; active violation escalates Nominal → Degraded; LockedOut never downgraded

**PARK-016 — RSS pre-actuator gate in KirraGovernor**
- Three-tier priority in `evaluate()`: LockedOut hard stop → RSS gate (Degraded semantics) → kinematic envelope
- `apply_mrc_profile()` extracted as single code path for Degraded and RSS-unsafe states
- `MRC_VELOCITY_CEILING_MPS` is the single source of truth — no bare `5.0` in source

**PARK-017 — RSS property tests**
- 3 blocks × 10,000 cases = 30,000 total proptest cases
- Block 1 Nominal: exact MRC contract when RSS unsafe; kinematic envelope when safe
- Block 2 Degraded: exact MRC regardless of RSS input
- Block 3 LockedOut: always 0.0 — hard stop dominates RSS gate

**PARK-018 — RssViolationEvent in audit chain**
- `append_rss_violation` follows static pattern on `AuditChainLinker`
- Chain hash input is event JSON — RSS events flow through `compute_record_hash` unchanged
- Test A: 5-entry mixed chain walks clean; Test B: one flipped bit fails hash check

**PARK-019 — 10,000-scenario adversarial trajectory simulation**
- 3 integration tests in `tests/rss_simulation.rs`; seeded `StdRng(0xDEAD_BEEF_CAFE)`
- `test_rss_adversarial_10k_scenarios`: 10,000 scenarios × 10 governor ticks; zero violations escaped; completes in 1.2 s
- `test_rss_posture_lifecycle_violation_to_recovery`: streak below threshold stays Degraded; threshold tick → Nominal
- `test_locked_out_hard_stop_dominates_rss_gate`: untrusted DAG → LockedOut; RSS-safe state does not override hard stop
