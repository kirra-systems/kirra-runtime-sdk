# Kirra Runtime SDK v1.5.0

**Release Date:** 2026-05-23
**Note:** This release was published under the previous "Aegis" branding. Safety documents use AEGIS-* identifiers. The rename to Kirra is tracked separately.

## Summary

v1.5.0 is the first release to include a multi-asset safety fabric, industrial protocol adapters, ROS2 safety interlock nodes, Ed25519-signed audit chain, a React operations dashboard, and a full ASIL-D safety case foundation. This release significantly expands the operational surface of the safety kernel from single-AV to multi-asset distributed deployments.

---

## Multi-Asset Safety Fabric

- Per-asset governors with isolated enforcement contexts
- Async routing of commands across fabric nodes
- Distributed telemetry aggregation with federated causal log
- Fleet lockout propagation across all registered assets (SG-007)
- Dashboard `FabricPage` component for fabric topology visualization

---

## ROS2 Safety Interlock

- `kirra_safety` ROS2 package (`cmd_vel_interceptor`, sensor monitor, posture subscriber)
- `cmd_vel` → governor → `cmd_vel_safe` pipeline wired to fleet posture
  <!-- Corrected 2026-06 (#171): the as-shipped topic is `/cmd_vel_safe`; the
       original note named `/filtered_cmd_vel`, which was never implemented. -->
- Sensor telemetry timeout detection (SG-003)
- Deploys on Ubuntu 24.04 + ROS2 Jazzy

---

## Industrial Protocol Adapters

Three new adapters with unified `/evaluate` endpoint integration:

- **EtherNet/IP** — Allen-Bradley PLC command interception
- **CANOpen** — NMT state change triggers posture recalculation (SG-011)
- **DNP3** — Broadcast commands mandatorily audit-logged (SG-012)

---

## Ed25519 Audit Chain Signing

- `AEGIS_LOG_SIGNING_KEY` environment variable (Ed25519 private key)
- All audit entries signed at append time; signature covers entry hash
- Tamper detection (SG-010): corrupt byte → signature verification failure

---

## React Operations Dashboard

- Fleet posture, node trust state, audit log viewer
- CORS support for cross-origin dashboard deployments
- Helm chart values expose `dashboard.enabled` toggle

---

## Deployment

- Docker multi-platform image: `linux/amd64`, `linux/arm64`, `linux/arm/v7`
- Helm chart for edge and cloud Kubernetes deployments
- CI release pipeline: automated build + SHA256SUMS artifact upload
- Binary assets: `x86_64-linux`, `aarch64-linux`, `armv7-linux`
